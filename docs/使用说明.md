## ⚙️ 配置说明

### 配置文件 config.json

```json
{
  "name": "纳斯达克100 ETF套利分析",
  "description": "分析纳斯达克100指数相关ETF的溢价率套利机会",
  "etf_codes": ["513390", "513300"],
  "threshold": 2.0,
  "default_days": 90,
  "benchmark": "纳斯达克"
}
```

**配置项说明**

| 配置项 | 必填 | 说明 | 示例 |
|--------|------|------|------|
| name | 否 | 分析任务名称 | "纳斯达克100 ETF套利分析" |
| description | 否 | 任务描述 | "分析..." |
| etf_codes | 是 | ETF代码列表 | ["513390", "513300"] |
| threshold | 否 | 套利阈值(%)，默认2.0 | 2.0 |
| default_days | 否 | 默认分析天数，默认90 | 90 |
| benchmark | 否 | 基准指数，默认"纳斯达克" | "纳斯达克"/"标普"/"日经" |

### 支持的基准指数

| 指数 | 配置值 | 说明 |
|------|--------|------|
| 纳斯达克100 | `纳斯达克` | 美股科技股指数 |
| 标普500 | `标普` | 美股大盘指数 |
| 日经225 | `日经` | 日本股市指数 |

## 🚀 使用流程

### 第一步：配置ETF代码

编辑 `config.json`，填入要分析的ETF代码：

```json
{
  "name": "我的ETF分析",
  "etf_codes": ["513100", "513390", "159941"],
  "threshold": 2.0,
  "benchmark": "纳斯达克"
}
```

### 第二步：运行分析

**方式一：完整分析（推荐）**
```bash
# 使用默认配置（config.json）
python etf_arbitrage.py

# 分析最近30天
python etf_arbitrage.py -d 30

# 使用自定义配置文件
python etf_arbitrage.py -c my_config.json
```

**方式二：仅生成可视化（基于已有数据）**
```bash
python visualize_interactive.py
```

### 第三步：查看报告

分析完成后，打开生成的报告：
```
data/report.html
```

用浏览器打开即可查看交互式图表。

## 📈 图表解读

### 1. 溢价率走势图
**用途**：观察ETF溢价率随时间的变化

**解读要点**：
- 溢价率 = (价格 - 净值) / 净值 × 100%
- 正值：价格高于净值（溢价）
- 负值：价格低于净值（折价）
- 寻找溢价率波动大的ETF进行套利

### 2. 跟踪误差走势图
**用途**：观察ETF跟踪指数的准确性

**解读要点**：
- 跟踪误差 = ETF净值涨幅 - 指数涨跌幅
- 接近0：跟踪精准
- 正值：ETF跑赢指数
- 负值：ETF跑输指数

### 3. 累计收益对比图
**用途**：对比ETF与指数的累计表现

**解读要点**：
- 从起始日归一化为0%
- 实线：ETF净值累计收益
- 虚线：指数累计收益
- 线与线越接近，跟踪越精准

### 4. 溢价率差值走势图
**用途**：寻找ETF间的套利机会

**解读要点**：
- 显示ETF对间的溢价率差值
- 差值 > 阈值：可能存在套利机会
- 红色点标记：识别到的套利机会

### 5. 溢价率分布图
**用途**：了解ETF溢价率的分布特征

**解读要点**：
- 直方图显示溢价率分布
- 正态分布说明价格稳定
- 分布越宽，套利机会越多

### 6. 套利统计表
**用途**：量化分析套利机会

**解读要点**：
- 点击表头可排序
- 关注"套利机会数"和"机会频率"
- 差值标准差大说明波动大

## 💡 使用建议

### 1. 如何选择ETF？
- 选择跟踪同一指数的不同ETF
- 优先选择流动性好的ETF
- 规模较大、交易活跃的ETF

### 2. 如何设置阈值？
- **保守型**：3-5%（机会少但安全）
- **平衡型**：2%（推荐）
- **激进型**：1%（机会多但风险高）

### 3. 套利策略建议
1. **观察期**：至少观察30天历史数据
2. **入场时机**：当溢价率差值 > 阈值时
3. **仓位管理**：单只ETF不超过总资金30%
4. **止损设置**：建议设置2-3%的止损线

### 4. 风险提示
⚠️ **重要提示**：
- 本工具仅供分析参考，不构成投资建议
- 溢价率回归可能需要时间，存在持仓风险
- 需考虑交易成本（佣金+滑点，约0.1-0.3%）
- 市场极端行情下套利策略可能失效

## 🔧 高级用法

### 自定义分析周期

```bash
# 分析最近30天
python etf_arbitrage.py -d 30

# 分析最近180天
python etf_arbitrage.py -d 180
```

### 多配置文件管理

创建多个配置文件用于不同场景：

```bash
# 纳斯达克ETF分析
config_nasdaq.json

# 日经ETF分析
config_nikkei.json

# 使用指定配置
python etf_arbitrage.py -c config_nikkei.json
```

### 清除缓存重新获取

如需重新获取数据，删除缓存文件：
```bash
# 删除所有缓存
rm -rf data/cache/*

# 删除特定ETF缓存
rm data/cache/513390.csv
```

## ❓ 常见问题

**Q1: 为什么某些ETF数据获取失败？**
- A: 可能是网络问题或该ETF代码不存在，请检查ETF代码是否正确

**Q2: 报告中的图表无法交互？**
- A: 请确保使用现代浏览器（Chrome/Firefox/Edge），并开启JavaScript

**Q3: 如何添加新的基准指数？**
- A: 目前支持纳斯达克、标普、日经三种指数，如需其他指数请修改代码中的BENCHMARK_CONFIGS

**Q4: 可以同时分析多少只ETF？**
- A: 建议不超过20只，数量过多会影响性能

**Q5: 数据更新频率如何？**
- A: 工具会自动缓存数据，每次运行时只获取新增数据

# ETF套利分析工具 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
ETF套利分析工具 (ETF Arbitrage Analysis Tool)

### 1.2 产品定位
为量化交易者和ETF投资者提供专业的ETF溢价率套利分析、跟踪误差分析和可视化报告生成工具。

### 1.3 目标用户
- ETF量化交易者
- 套利策略研究员
- 基金投资分析师
- 个人专业投资者

## 2. 核心功能需求

### 2.1 数据采集模块
- **ETF数据获取**: 通过AKShare获取ETF历史价格、净值(IOPV)数据
- **指数数据获取**: 支持获取纳斯达克100、标普500、日经225等全球主要指数数据
- **智能缓存**: 自动缓存已获取的数据，避免重复请求，支持增量更新

### 2.2 数据分析模块
- **溢价率计算**: 计算每日ETF溢价率 (价格-净值)/净值 × 100%
- **跟踪误差计算**: 计算ETF净值涨幅与基准指数涨跌幅的差异
- **跟踪精度计算**: 基于R²决定系数评估ETF跟踪质量
- **套利机会识别**: 识别ETF间溢价率差值超过阈值的套利机会

### 2.3 可视化模块
- **溢价率走势图**: 多ETF溢价率对比时间序列图
- **跟踪误差走势图**: ETF相对于基准指数的跟踪偏离度
- **累计收益对比图**: ETF与基准指数的归一化累计收益对比
- **溢价率差值走势图**: 多组ETF配对间的溢价率差值
- **溢价率分布图**: 各ETF溢价率分布直方图
- **交互式报告**: 生成包含所有图表的HTML交互式报告

### 2.4 报告生成模块
- **套利统计表**: 统计最大/平均差值、套利机会次数、频率等
- **跟踪精度排名表**: 基于R²系数的ETF跟踪质量排名
- **可排序表格**: 支持点击表头排序的交互式表格

## 3. 技术架构

### 3.1 技术栈
- **Python**: 核心编程语言
- **AKShare**: 金融数据获取库
- **Pandas/NumPy**: 数据处理与分析
- **Plotly**: 交互式图表可视化
- **HTML/CSS/JavaScript**: 报告前端展示

### 3.2 模块结构
```
etf_workspace/
├── etf_arbitrage.py          # 核心数据获取与分析模块
├── visualize_interactive.py  # 可视化与报告生成模块
├── config.json               # 配置文件
├── data/                     # 数据存储目录
│   ├── cache/               # 缓存数据
│   ├── *_premium.csv        # ETF溢价率数据
│   ├── *_tracking_error.csv # 跟踪误差数据
│   ├── report.html          # 生成的报告
│   └── sort-table.js        # 表格排序脚本
└── docs/                     # 文档目录
```

## 4. 配置需求

### 4.1 配置文件结构
```json
{
  "name": "分析任务名称",
  "description": "分析描述",
  "etf_codes": ["ETF代码列表"],
  "threshold": 2.0,
  "default_days": 90,
  "benchmark": "基准指数名称"
}
```

### 4.2 支持的基准指数
- **纳斯达克**: 纳斯达克100指数 (.NDX)
- **标普**: 标普500指数 (.INX)
- **日经**: 日经225指数

## 5. 性能需求

### 5.1 数据处理性能
- 单次分析支持最多20只ETF
- 历史数据查询默认90天，可自定义
- 数据缓存机制减少API调用

### 5.2 响应时间
- 数据获取: < 30秒（含缓存）
- 报告生成: < 10秒

## 6. 扩展性需求

### 6.1 数据源扩展
- 预留接口支持添加新的指数数据源
- 支持不同数据提供商的切换

### 6.2 分析维度扩展
- 支持自定义分析指标
- 支持自定义图表类型

## 7. 安全与合规

### 7.1 数据安全
- 仅使用公开市场数据
- 不涉及交易执行
- 本地缓存不存储敏感信息

### 7.2 免责声明
- 工具仅供分析参考
- 不构成投资建议
- 用户需自行承担投资风险

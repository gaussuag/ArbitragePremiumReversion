# ETF套利分析工具 - 功能清单

## 📊 核心功能模块

### 1. 数据采集功能

| 功能名称 | 功能描述 | 实现状态 | 技术实现 |
|---------|---------|---------|---------|
| ETF价格数据获取 | 获取ETF历史交易价格 | ✅ 已实现 | AKShare: fund_etf_hist_em |
| ETF净值数据获取 | 获取ETF每日净值(IOPV) | ✅ 已实现 | AKShare: fund_etf_hist_em |
| 指数数据获取 | 获取基准指数历史数据 | ✅ 已实现 | AKShare: index_us_stock_sina / index_global_hist_sina |
| 智能数据缓存 | 自动缓存已获取数据，支持增量更新 | ✅ 已实现 | CSV文件缓存 |
| 缓存管理 | 支持缓存加载、合并、去重 | ✅ 已实现 | ETFCacheManager类 |

### 2. 数据分析功能

| 功能名称 | 功能描述 | 计算公式 | 实现状态 |
|---------|---------|---------|---------|
| 溢价率计算 | 计算ETF每日溢价率 | `(价格-净值)/净值 × 100%` | ✅ 已实现 |
| 跟踪误差计算 | 计算ETF与指数的跟踪差异 | `ETF净值涨幅 - 指数涨跌幅` | ✅ 已实现 |
| 跟踪精度计算 | 基于R²决定系数评估跟踪质量 | `correlation²` | ✅ 已实现 |
| 套利机会识别 | 识别ETF对间的套利机会 | `溢价率差值 ≥ 阈值` | ✅ 已实现 |
| 统计指标计算 | 计算均值、标准差、最大/最小值等 | 标准统计方法 | ✅ 已实现 |

## 🔧 配置功能

### 支持的配置项

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| name | 分析任务名称 | "纳斯达克100 ETF套利分析" |
| description | 分析描述 | "分析..." |
| etf_codes | ETF代码列表 | ["513390", "513300"] |
| threshold | 套利阈值(%) | 2.0 |
| default_days | 默认分析天数 | 90 |
| benchmark | 基准指数 | "纳斯达克" / "标普" / "日经" |

### 支持的基准指数

| 指数名称 | 配置值 | 代码 | 数据源 |
|---------|--------|------|--------|
| 纳斯达克100 | 纳斯达克 | .NDX | AKShare美股接口 |
| 标普500 | 标普 | .INX | AKShare美股接口 |
| 日经225 | 日经 | 日经225指数 | AKShare全球指数接口 |

## 🚀 命令行功能

| 命令 | 功能 | 示例 |
|------|------|------|
| `python etf_arbitrage.py` | 执行完整分析流程 | 使用config.json默认配置 |
| `python etf_arbitrage.py -c config_custom.json` | 使用自定义配置文件 | 指定其他配置文件 |
| `python etf_arbitrage.py -d 30` | 分析最近N天 | 分析最近30天 |
| `python visualize_interactive.py` | 仅生成可视化报告 | 基于已有数据生成报告 |

## 📈 统计指标

### 单只ETF统计

- 平均溢价率
- 最高/最低溢价率
- 溢价率标准差（波动率）
- 平均跟踪误差
- 最大跟踪误差
- 跟踪精度(R²)

### ETF配对统计

- 共同交易日数量
- 最大正差值
- 最大负差值
- 平均差值
- 标准差
- 套利机会次数
- 套利机会频率
